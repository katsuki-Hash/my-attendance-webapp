アプリ名（勤怠管理システム）
■ 概要
Firebaseを用いて開発した勤怠管理Webアプリケーションです。
ユーザー認証・打刻・勤怠管理・権限管理まで一通りの機能を実装しています。

■ 使用技術
・フロントエンド：HTML / JavaScript　/　CSS
・バックエンド：Firebase
  * Authentication（ユーザー認証）
  * Firestore（データベース）
  * Firebase Hosting（デプロイ）
・開発環境：GitHub / Codespaces（VSCode）
・バージョン管理：GitHub

■ 機能一覧

・ユーザー登録 / ログイン機能
　メール / パスワードによる登録・ログイン・ログアウト機能
・勤怠打刻
　出勤、遅刻の打刻　/　同日のデータを1レコードとして管理
・勤怠履歴一覧表示
　ユーザーごとの勤怠履歴を一覧表示
・月次集計
　出勤・退勤時間から勤務時間を自動計算（勤務時間 = 退勤 - 出勤）
・権限管理（RBAC）
　一般ユーザー：自分の勤怠のみ閲覧可能
　管理者：全ユーザーの勤怠を閲覧可能
　Firebase Security Rulesでアクセス制御を実装
・バリデーション
　重複打刻防止
　出勤前の退勤防止
・CSVダウンロード機能
　勤怠データをCSV形式で出力可能

■ 工夫した点

① ユーザーごとのデータ分離
Firebase AuthenticationのUIDを用いて、ユーザーごとの勤怠データを管理しました。

② 権限管理の実装
Firestoreにユーザーのrole（admin / user）を保持し、
Firebase Security Rulesでアクセス制御を行うことで、RBAC（ロールベースアクセス制御）を実現しました。

③ バリデーション処理
業務を想定し、不正な打刻（重複出勤・未出勤での退勤）を防ぐロジックを実装しました。

④ 実務を意識した機能
CSVダウンロード機能を実装し、実務でのデータ活用を想定しました。
